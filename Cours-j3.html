📝 Prérequis

Les Jours 1 & 2 : vous savez lire un MLD (tables, clés primaires 🔑, clés étrangères #). Ici, ce MLD devient une vraie base PostgreSQL que l'on interroge. Côté machine : Windows avec les droits administrateur.

Fil rouge : une base bibliothèque — auteurs, éditeurs, livres, adhérents, emprunts. Tout est préparé dans un conteneur : vous n'aurez qu'à le lancer.

Le schéma de la base « bibliothèque »

Avant d'écrire une requête, il faut connaître le terrain. Voici le MCD, puis le MLD qui a servi à créer les tables.

Le MCD — le plan

🪄 Rappel métaphore

Le MCD reste le plan d'architecte du Jour 2 : des entités (rectangles), des associations (ovales, nommées par un verbe) et des cardinalités. Voici celui de notre bibliothèque.

À lire simplement : un livre est écrit par un ou plusieurs auteurs (n-n), il est publié par exactement un éditeur, et il peut apparaître dans plusieurs emprunts. Chaque emprunt concerne un seul livre et est effectué par un seul adhérent.

Le MLD — les tables

Installer Docker sur Windows (via WSL2)

🪄 La métaphore

Docker, c'est un conteneur maritime préemballé. À l'intérieur : la base PostgreSQL, sa version exacte, sa configuration, les données. On expédie la boîte ; elle s'ouvre à l'identique sur n'importe quelle machine. Fini le « ça marche chez moi ».

Définition. Docker fait tourner des applications dans des conteneurs isolés. Sur Windows, Docker s'appuie sur WSL2 (« Windows Subsystem for Linux », c'est-à-dire « sous-système Linux pour Windows »), un mini-Linux intégré à Windows.

Étape 1 — Installer WSL2

Ouvrez PowerShell en administrateur (clic droit sur le menu Démarrer → « Terminal (administrateur) ») et tapez :

wsl --install

Cette commande installe WSL2 et une distribution Ubuntu par défaut.

⚠️ Redémarrage obligatoire

Après wsl --install, il faut redémarrer le PC pour finaliser. Au retour, Ubuntu peut demander de créer un identifiant/mot de passe Linux : c'est normal.

Pour vérifier que WSL2 est bien là :

wsl --status
wsl --list --verbose   # la colonne VERSION doit afficher 2

Étape 2 — Installer Docker Desktop

Toujours dans PowerShell :

winget install -e --id Docker.DockerDesktop

(Ou téléchargez « Docker Desktop for Windows » depuis le site officiel, puis lancez l'installeur.)

  1. Lancez Docker Desktop une première fois et laissez-le démarrer (icône baleine dans la barre des tâches).
  2. Dans Settings → Resources → WSL integration, vérifiez que l'intégration WSL2 est activée.

Vérifiez enfin que Docker répond :

docker --version
docker run hello-world   # test : télécharge et lance un mini-conteneur
💡 Docker Desktop doit tourner

Tant que la baleine Docker Desktop n'est pas démarrée, aucune commande docker ne fonctionne. Gardez-la ouverte pendant tout le TP.

Lancer le conteneur de la base

On vous fournit deux fichiers à mettre dans un même dossier :

📄docker-compose.yml— la recette du conteneur PostgreSQL
📄bibliotheque_seed.sql— le schéma + les ~230 lignes de données

Le fichier docker-compose.yml décrit tout :

services:
  bibliotheque-db:
    image: postgres:16
    container_name: bibliotheque
    environment:
      POSTGRES_USER: root
      POSTGRES_PASSWORD: root
      POSTGRES_DB: bibliotheque
    ports:
      - "5432:5432"   # hote:conteneur
    volumes:
      - ./bibliotheque_seed.sql:/docker-entrypoint-initdb.d/01_seed.sql:ro

Placez-vous dans le dossier (dans un terminal) et lancez :

cd chemin/vers/le/dossier
docker compose up -d      # -d = en arriere-plan (detached)

Vérifiez que le conteneur tourne :

docker ps                 # doit lister "bibliotheque"
docker compose logs -f    # suit le demarrage (Ctrl+C pour quitter)
🎯 Le mock se charge tout seul

Au tout premier démarrage, PostgreSQL exécute automatiquement tout fichier .sql placé dans /docker-entrypoint-initdb.d/. Notre bibliotheque_seed.sql y est monté : la base est donc déjà remplie quand le conteneur est prêt. (Voir §5 si vous devez la recharger.)

⚠️ Le port 5432 est déjà pris ?

Si un PostgreSQL est déjà installé sur la machine, le port 5432 est occupé. Changez la partie gauche du mapping, par exemple "5433:5432", puis connectez-vous sur le port 5433.

Se connecter avec DBeaver

🪄 La métaphore

DBeaver est le guichet depuis lequel vous remettez vos « bons de commande » (les requêtes SQL) au magasinier (le SGBD PostgreSQL). Le conteneur est le magasin ; DBeaver est le comptoir.

Dans DBeaver : Base de données → Nouvelle connexion → PostgreSQL, puis remplissez les champs.

Champ DBeaverValeurD'où ça vient
Host / ServeurlocalhostLe conteneur publie son port sur votre machine
Port5432La partie gauche de "5432:5432"
DatabasebibliothequeLa variable POSTGRES_DB
UsernamerootLa variable POSTGRES_USER
PasswordrootLa variable POSTGRES_PASSWORD
  1. Si DBeaver propose de télécharger le driver PostgreSQL, acceptez.
  2. Cliquez Test Connection. Si c'est vert : Finish.
⚠️ Où sont les tables ? (schéma public)

En PostgreSQL, les tables vivent dans un schéma, public par défaut. Dans l'arbre DBeaver, dépliez bibliotheque → Schemas → public → Tables. Si vous ne voyez pas d'autres bases, activez au besoin Show all databases dans les réglages de la connexion (onglet PostgreSQL).

💡 Écrire ses requêtes

Une fois connecté, l'onglet SQL Editor (icône ou Ctrl+]) sert à taper vos requêtes ; Ctrl+Entrée exécute la requête courante.

Peupler la base avec le mock

Avec le docker-compose.yml ci-dessus, rien à faire : le mock est chargé au premier démarrage. Ce jeu de données contient environ 230 lignes :

TableLignes

Vérifiez que tout est là (dans le SQL Editor de DBeaver) :

SELECT COUNT(*) FROM livre;    -- doit renvoyer 50

Recharger la base à la main (si besoin)

Si vous devez rejouer le mock (base modifiée, exercice raté…), deux options.

Option A — repartir de zéro (efface tout, recharge le seed au démarrage) :

docker compose down -v    # -v supprime le volume de données
docker compose up -d

Option B — rejouer le fichier dans le conteneur déjà lancé (le seed supprime puis recrée les tables, il est rejouable) :

docker exec -i bibliotheque psql -U root -d bibliotheque < bibliotheque_seed.sql
💡 Ou tout simplement dans DBeaver

Ouvrez le fichier bibliotheque_seed.sql dans DBeaver (SQL Editor → ouvrir un script) et exécutez-le en entier avec Alt+X.

Les commandes SQL de base

🪄 La métaphore filée

Une requête SQL est un bon de commande remis au bibliothécaire. SELECT dit « apporte-moi… », FROM précise « dans quel rayon », WHERE pose les critères, ORDER BY l'ordre de présentation.

SELECT … FROM — tout lire

Syntaxe.

SELECT * FROM nom_de_la_table;

Exemple.

SELECT * FROM livre;

Analysons ligne par ligne : SELECT * demande toutes les colonnes (l'étoile = « tout ») ; FROM livre indique la table ; le ; termine la requête.

⚠️ N'oubliez pas le point-virgule

Chaque requête se termine par ;. C'est ce qui dit au SGBD « j'ai fini, exécute ».

Choisir les colonnes

SELECT titre, annee_publication FROM livre;
💡 Bonne pratique

En vrai projet, on évite SELECT * : on nomme les colonnes utiles. C'est plus lisible et plus rapide.

WHERE — filtrer les lignes

SELECT titre, genre FROM livre
WHERE genre = 'Roman';

Opérateurs utiles : = (égal), <> (différent), < > <= >=, et les combinateurs AND / OR.

SELECT titre, annee_publication, nb_pages FROM livre
WHERE annee_publication > 1950 AND nb_pages < 300;
⚠️ Deux pièges classiques

1. En SQL, l'égalité s'écrit avec un seul = (pas == comme en Java).
2. Le texte va entre apostrophes simples : 'Roman', jamais entre guillemets doubles.

ORDER BY et LIMIT — trier, limiter

SELECT titre, annee_publication FROM livre
ORDER BY annee_publication DESC
LIMIT 10;

DESC = décroissant (« descending »), ASC = croissant (par défaut). LIMIT 10 ne garde que les 10 premières lignes.

DISTINCT et LIKE

-- la liste des genres, sans doublon
SELECT DISTINCT genre FROM livre;

-- les titres qui commencent par "L"
SELECT titre FROM livre WHERE titre LIKE 'L%';

Le % de LIKE remplace « n'importe quelle suite de caractères ».

Compter et calculer — les fonctions d'agrégation

SELECT COUNT(*)      FROM emprunt;   -- combien d'emprunts ?
SELECT AVG(nb_pages) FROM livre;     -- nombre de pages moyen
SELECT MIN(annee_publication), MAX(annee_publication) FROM livre;

Principales fonctions : COUNT (compter), SUM (somme), AVG (moyenne), MIN, MAX.

GROUP BY — regrouper puis compter

-- combien de livres par genre ?
SELECT genre, COUNT(*) AS nb
FROM livre
GROUP BY genre
ORDER BY nb DESC;

GROUP BY genre range les livres par genre ; COUNT(*) compte dans chaque groupe ; AS nb renomme la colonne du résultat.

JOIN — recoller deux tables

🪄 La métaphore

Un JOIN, c'est suivre le renvoi (§Jour 2). La fiche livre contient #id_editeur : le JOIN va chercher la fiche editeur correspondante et recolle les deux.

-- chaque livre avec le nom de son editeur
SELECT livre.titre, editeur.nom
FROM livre
JOIN editeur ON livre.id_editeur = editeur.id_editeur;

La clause ON dit sur quelle égalité coller : la clé étrangère d'un côté = la clé primaire de l'autre.

Pour la relation n-n (livre ↔ auteur), on passe par la table associative ecrire, avec deux JOIN :

-- chaque livre avec le(s) nom(s) de son/ses auteur(s)
SELECT livre.titre, auteur.nom
FROM livre
JOIN ecrire ON livre.id_livre  = ecrire.id_livre
JOIN auteur ON ecrire.id_auteur = auteur.id_auteur;

INSERT, UPDATE, DELETE — écrire dans la base

-- ajouter un auteur
INSERT INTO auteur (nom, prenom, nationalite, annee_naissance)
VALUES ('Verne', 'Jules', 'Francaise', 1828);

-- corriger un genre
UPDATE livre SET genre = 'Aventure' WHERE id_livre = 25;

-- supprimer un emprunt
DELETE FROM emprunt WHERE id_emprunt = 12;
⚠️ Le piège le plus dangereux

Un UPDATE ou un DELETE sans WHERE s'applique à toute la table. DELETE FROM emprunt; vide tous les emprunts. Vérifiez toujours votre WHERE avant d'exécuter.

🎯 Réflexes SQL

🎯 Les 5 réflexes du débutant
  1. Toujours finir par ;.
  2. Égalité = un seul = ; texte entre apostrophes simples '…'.
  3. Jamais de UPDATE/DELETE sans WHERE (sauf intention réelle de tout toucher).
  4. Un JOIN se fait sur une clé (FK = PK) via ON.
  5. Docker Desktop doit tourner pour que docker réponde.
